## 一、系统架构概述

### 1.1 整体架构

本软件采用Unity/Tuanjie Engine 2D URP架构，使用C#进行开发，遵循以下分层架构设计：

```
┌─────────────────────────────────────────────┐
│              UI Layer (界面层)               │
│  HomeUI / GenderSelectUI / HighSchoolUI     │
│  CollegeUI / LifeUI / ResultUI              │
├─────────────────────────────────────────────┤
│           Manager Layer (管理器层)           │
│  GameManager / GameStateManager / SaveManager│
│  AdManager / ShareManager / PlatformManager │
├─────────────────────────────────────────────┤
│            Engine Layer (引擎层)             │
│  EventEngine / TalentEngine / ScoreEngine   │
│  PersonalityEngine / DejaVuEngine            │
│  InheritEngine / MergeEngine / LifeEngine   │
├─────────────────────────────────────────────┤
│             Data Layer (数据层)             │
│  DataLoader / ResourceHelper                │
├─────────────────────────────────────────────┤
│            Model Layer (模型层)              │
│  PlayerState / GameEvent / Talent           │
│  College / Major / Province / NPC            │
└─────────────────────────────────────────────┘
```

### 1.2 核心设计模式

- **单例模式(Singleton)**：所有Manager和Engine类使用Instance单例模式，确保全局唯一
- **观察者模式(Observer)**：GameManager通过event实现状态变更通知
- **工厂模式**：DataLoader作为数据工厂统一创建各类游戏数据对象
- **策略模式**：事件系统支持多种交互策略（choice/merge/sort/slider/timing/dialog）

## 二、核心模块设计

### 2.1 游戏管理器（GameManager）

游戏总控制器，负责游戏生命周期管理。

**主要职责：**
- 管理游戏全局状态（isGameRunning, isPaused）
- 协调各Engine和Manager之间的通信
- 处理场景切换逻辑
- 管理玩家数据读写

**核心方法：**
- `StartNewGame()`：初始化新游戏流程
- `ContinueGame()`：从存档恢复游戏
- `ChangePhase(GamePhase)`：阶段切换
- `SaveGame(slotIndex)`：保存游戏数据
- `CalculateGaokaoScore()`：调用ScoreEngine计算高考成绩
- `StartNewGamePlus()`：二周目新游戏创建

### 2.2 游戏状态管理器（GameStateManager）

独立于GameManager的状态管理组件，生命周期贯穿所有场景。

**主要职责：**
- 维护GameSaveData数据结构
- 提供QuickSave/QuickLoad快速存取接口
- 管理玩家性别（Gender）信息
- 协调存档与引擎间的数据同步

**与GameManager的区别：**
- GameManager专注游戏流程控制
- GameStateManager专注数据持久化

### 2.3 事件引擎（EventEngine）

驱动游戏进程的核心引擎，负责事件的获取、选择和结算。

**核心流程：**
```
获取事件 → 检查触发条件 → 检查既视感 → 展示事件
    ↓
玩家交互选择 → 调用ResolveEvent计算结果 → 更新属性
    ↓
记录事件完成 → 获取下一事件 → 循环
```

**核心方法：**
- `SetPlayerState(PlayerState)`：注入玩家状态
- `GetNextEvent()`：获取下一事件
- `ResolveEvent(eventId, choiceId)`：结算事件结果
- `GetWeightedRandom()`：加权随机选择

### 2.4 天赋引擎（TalentEngine）

管理12种天赋的抽卡、选择和效果计算。

**核心方法：**
- `DrawTalents(int count)`：抽卡获取候选天赋
- `SelectTalent(string talentId)`：玩家选择天赋
- `GetCurrentTalent()`：获取当前天赋
- `ApplyTalentEffect(PlayerState)`：应用天赋效果
- `GetTalentBonus(string statName)`：获取天赋对特定属性的加成

### 2.5 成绩计算引擎（ScoreEngine）

根据玩家属性和天赋计算高考成绩。

**计算公式：**
```
最终成绩 = 智力基础分 × 0.7 + 心态影响分 × 0.2 + 天赋加成 + 省份难度调整
```

**核心方法：**
- `CalculateScore(PlayerState, string talentId)`：主计算入口
- `GetSubjectScore(subject, baseStat)`：计算单科成绩
- `ApplyProvinceModifier(score, province)`：应用省份难度系数

### 2.6 人格引擎（PersonalityEngine）

根据玩家属性计算高考人格类型。

**人格判定规则：**
- 取Intellect/Mental/Social/Health四个维度的最高值
- 若存在多个维度相等，取优先级最高的维度对应人格
- 人格类型决定游戏内部分对话和结局倾向

### 2.7 既视感引擎（DejaVuEngine）

二周目记忆系统，提供前世记忆和选择提示。

**既视感等级（L1-L4）：**
- L1：微弱直觉，仅在关键选项时显示模糊提示
- L2：记忆片段，显示选项可能导致的属性变化方向
- L3：清晰记忆，显示每个选项的历史结果
- L4：完美直觉，解锁隐藏剧情选项

### 2.8 传承引擎（InheritEngine）

多代传承系统，将当前人生的属性遗传给下一代。

**传承内容：**
- 属性加成（Intellect/Mental/Social/Health各传承10-15%）
- 记忆天赋（既视感等级）
- 校友院校（解锁特定招生优惠）

## 四、资源加载设计

### 4.1 ResourceHelper 资源加载策略

采用"先加载后降级"的策略：
1. 优先从Resources目录加载指定路径的Sprite
2. 若加载失败，自动生成彩色占位Texture2D
3. 将Texture2D动态创建为Sprite返回

**性别感知加载：**
```csharp
public static Sprite LoadEventIllustration(string illustrationId, Gender gender, string stage)
{
    string prefix = gender == Gender.Male ? "男illu" : "女illu";
    string path = $"Tex/{stageFolder}/{prefix}_{illustrationId}";
    return LoadSprite(path, 680, 380, new Color(0.9f, 0.9f, 0.95f));
}
```

## 五、存档系统设计

### 5.1 存档策略

采用PlayerPrefs作为本地存储媒介：
- 存档键：`gaokao_life_save`
- 数据格式：JSON字符串
- 自动保存：关键节点自动调用QuickSave

### 5.2 存档时机

- 事件完成后自动保存
- 阶段切换时保存
- 应用退到后台时保存
- 手动点击保存按钮时保存

### 5.3 存档数据结构

```csharp
public class GameSaveData
{
    public string saveId;            // UUID唯一标识
    public long createdAt;           // Unix时间戳
    public long updatedAt;           // 最后更新时间戳
    public string currentStage;      // 当前阶段字符串
    public PlayerState playerState; // 玩家完整状态
    public InheritData inheritData;  // 传承数据
    public List<PlaythroughMemory> pastMemories; // 前世记忆列表
}
```

## 六、多平台适配设计

### 6.1 平台检测

使用条件编译指令区分不同平台：
```csharp
#if UNITY_WECHAT_GAME
    // 微信小游戏特定代码
#elif UNITY_EDITOR
    // 编辑器特定代码
#else
    // 其他平台
#endif
```

### 6.2 分辨率适配

- 设计分辨率：750×1334（9:16竖屏）
- Canvas缩放模式：Scale With Screen Size
- 参考分辨率：宽度750作为基准

### 6.3 构建配置

通过BuildConfig编辑器工具配置各平台参数：
- 微信小游戏：内存限制、首包大小、图形API
- 抖音小游戏：WebGL构建配置
- TapTap：Android/iOS包配置

## 七、安全性设计

### 7.1 数据校验

- 所有存档数据使用JSON序列化，包含完整性校验
- 属性值通过Clamp限制在0-100范围内
- 枚举值在反序列化时进行有效性检查

### 7.2 存档保护

- 存档数据存储于应用私有目录
- 不在日志中输出敏感数据
- 版本升级时提供存档迁移机制
